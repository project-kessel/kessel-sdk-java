# Testing Guidelines

Conventions and patterns for writing tests in the kessel-sdk-java repository.

## Build and Toolchain

- **Java 21**, **JUnit Jupiter 6.0.1**, **Mockito 5.21.0** (core) / **5.20.0** (junit-jupiter), **grpc-testing** (matches `grpc.version` property: 1.77.0).
- Surefire plugin (`3.5.4`) picks up files matching `**/*Test.java` and `**/*Tests.java`.
- Autogenerated protobuf classes under `v1`, `v1beta1`, and select `v1beta2` files are excluded from Surefire. Never place hand-written tests in those packages.
- CI runs `./mvnw clean verify` on JDK 21 (Ubuntu). All tests must pass in that pipeline before merge.

## File Organization

| Source location | Test location |
|---|---|
| `kessel-sdk/src/main/java/org/project_kessel/api/auth/` | `kessel-sdk/src/test/java/org/project_kessel/api/auth/` |
| `kessel-sdk/src/main/java/org/project_kessel/api/inventory/` | `kessel-sdk/src/test/java/org/project_kessel/api/inventory/` |
| `kessel-sdk/src/main/java/org/project_kessel/api/rbac/v2/` | `kessel-sdk/src/test/java/org/project_kessel/api/rbac/v2/` |

Rules:
1. Mirror the source package path exactly in `src/test/java`.
2. Name the test class `<ClassName>Test` (e.g., `Utils` -> `UtilsTest`).
3. Use package-private visibility on test classes (no `public` modifier).

## Test Class Structure

Every test class follows this template:

```java
@ExtendWith(MockitoExtension.class)  // omit if no mocks needed (see UtilsTest)
class FooTest {

    @Mock
    private SomeDependency mockDep;    // prefix with "mock"

    @BeforeEach
    void setUp() { /* shared setup */ }

    @Test
    void testDescriptiveName() { ... }
}
```

Rules:
1. Annotate `@ExtendWith(MockitoExtension.class)` when using `@Mock` fields or Mockito verification APIs. May be omitted when no mocking is needed (see `UtilsTest`).
2. Use `@BeforeEach void setUp()` for shared initialization. Do not use `@BeforeAll`.
3. No test class uses constructor injection or `@InjectMocks` -- prefer explicit wiring.

## Test Method Naming

Pattern: `test<What>` in camelCase, describing the scenario.

Examples from the codebase:
- `testConstructorWithValidConfig`
- `testConstructorWithNullTarget`
- `testFetchRootWorkspaceWithNullEndpoint`
- `testHandlesPagination`
- `testStopsWhenNoToken`
- `testBuildCreatesStub`
- `testReturnsWorkspacesAcrossMultiplePages`

Rules:
1. Start every method name with `test`.
2. Include the behavior or condition being verified after `test`.
3. For null/edge cases, suffix with the condition: `WithNull...`, `WithEmpty...`, `WithInvalid...`.

## Mocking Patterns

### Mockito `@Mock` Fields
Declare mock dependencies as fields with `@Mock`:

```java
@Mock private HttpClient mockHttpClient;
@Mock private HttpResponse<byte[]> mockResponse;
@Mock private AuthRequest mockAuth;
```

Naming: prefix with `mock`.

### gRPC Stub Mocking
Mock the generated `BlockingStub` directly -- do not use `grpc-testing` InProcessServer for unit tests:

```java
@Mock
private KesselInventoryServiceBlockingStub mockInventoryClient;

when(mockInventoryClient.streamedListObjects(any(StreamedListObjectsRequest.class)))
        .thenReturn(firstPageIterator)
        .thenReturn(secondPageIterator);
```

For streaming responses, return `Iterator<ResponseType>` built from `List.of(...).iterator()` or `Collections.emptyIterator()`.

### HttpClient Mocking (REST calls)
Mock `HttpClient` and `HttpResponse` separately. Use `lenient()` in `@BeforeEach` for the generic send stub, then override `statusCode()` and `body()` per test:

```java
@SuppressWarnings("unchecked")
@BeforeEach
void setUp() throws IOException, InterruptedException {
    lenient().when(mockHttpClient.send(any(HttpRequest.class), any(HttpResponse.BodyHandler.class)))
            .thenReturn(mockResponse);
}
```

### ArgumentCaptor for Request Verification
Use `@Captor` fields to capture and inspect gRPC request objects:

```java
@Captor
private ArgumentCaptor<StreamedListObjectsRequest> requestCaptor;

verify(mockInventoryClient, times(2)).streamedListObjects(requestCaptor.capture());
List<StreamedListObjectsRequest> requests = requestCaptor.getAllValues();
assertEquals("next-page-token", requests.get(1).getPagination().getContinuationToken());
```

### Abstract Class Testing
Create concrete inner test implementations of abstract classes within the test file:

```java
static class TestClientBuilder extends AbstractClientBuilder<TestStub, TestAsyncStub> {
    TestClientBuilder(String target) { super(target); }

    @Override
    protected TestStub newStub(Channel channel) { return new TestStub(channel); }
    @Override
    protected TestAsyncStub newAsyncStub(Channel channel) { return new TestAsyncStub(channel); }
}
```

These inner classes are `static` and package-private.

## Assertion Patterns

Use only `org.junit.jupiter.api.Assertions` static imports. The repo does not use AssertJ or Hamcrest.

| Pattern | Usage |
|---|---|
| `assertEquals(expected, actual)` | Value comparison (most common) |
| `assertNotNull(value)` | Existence checks |
| `assertNull(value)` | Null state verification |
| `assertTrue(condition)` / `assertFalse(condition)` | Boolean checks, `hasXxx()` protobuf accessors |
| `assertSame(expected, actual)` | Builder fluent-API identity checks |
| `assertThrows(ExType.class, () -> ...)` | Exception verification |
| `assertDoesNotThrow(() -> ...)` | Verify no exception on valid input |

Rules:
1. For exception messages, capture the exception and assert with `assertTrue(e.getMessage().contains("..."))`.
2. Use `assertThrows` for null-argument validation (expect `NullPointerException`).
3. Use `assertSame` (not `assertEquals`) to verify builder methods return `this`.

## Protobuf Test Data

Build protobuf test objects using the generated `.newBuilder()` pattern. Create private static helper methods for reusable test data:

```java
private static StreamedListObjectsResponse responseWithToken(String token) {
    return StreamedListObjectsResponse.newBuilder()
            .setPagination(ResponsePagination.newBuilder()
                .setContinuationToken(token).build())
            .build();
}
```

For JSON test data (HTTP responses), use text blocks as `static final String` constants:

```java
private static final String VALID_ROOT_WORKSPACE_JSON = """
        {
            "data": [{ "id": "root-ws-1", "name": "Root Workspace", "type": "root" }]
        }
        """;
```

## What to Test

The repo tests the following categories -- follow the same coverage pattern:

1. **Null-argument rejection**: Every public method that calls `Objects.requireNonNull` gets a corresponding `assertThrows(NullPointerException.class, ...)` test.
2. **Builder fluent API**: Verify each builder method returns the same instance (`assertSame`), and that `build()`/`buildAsync()` produce the correct stub types. Shut down `ManagedChannel` in cleanup.
3. **Pagination logic**: Test first page, multi-page, empty results, initial continuation token, and stop-on-empty-token.
4. **Error propagation**: Verify that gRPC/HTTP errors are caught and re-thrown with descriptive messages.
5. **Record/POJO accessors**: Verify getters return expected values, including `equals`/`hashCode` for records.
6. **Edge cases for external dependencies**: When testing code that depends on optional libraries (Nimbus OAuth), use `Class.forName` checks or try/catch to skip gracefully.

## Resource Cleanup

When tests call `builder.build()` or `builder.buildAsync()`, always shut down the `ManagedChannel`:

```java
Pair<TestStub, ManagedChannel> result = builder.build();
// assertions...
result.getRight().shutdown();
```

## Anti-Patterns to Avoid

- Do not use `@InjectMocks` -- this repo prefers explicit construction.
- Do not use `grpc-testing` `InProcessServer` for unit tests -- mock the blocking stub directly.
- Do not add AssertJ or Hamcrest -- use JUnit Jupiter assertions only.
- Do not use `@BeforeAll` or shared mutable static state.
- Do not make test classes or methods `public` -- use package-private.
- Do not place test code in protobuf-generated packages that Surefire excludes.
